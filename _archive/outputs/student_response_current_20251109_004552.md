class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

def find_number(head, target):
    current = head
    position = 0
    
    while current is not None:
        if current.data == target:
            return position
        current = current.next
        position += 1
    
    return -1  # Not found

# Example usage
node1 = Node(10)
node2 = Node(20)
node3 = Node(30)
node1.next = node2
node2.next = node3

print(find_number(node1, 20))  # Returns 1
print(find_number(node1, 40))  # Returns -1

In this case, the map is like a variable holding a reference to location "A". The arrow pointing to "B" is like the `next` attribute in our Node class.

Now, let's say we want to move the map to point directly to location "B". We can do this by reassigning the variable to hold a reference to location "B". This is equivalent to doing `current = current.next`.

Here's a step-by-step diagram to illustrate this:

**Before:**
- `current` → node1
- `node1.next` → node2

**To move `current` to point to node2:**
1. We keep the value of `node1.next`, which is a reference to node2.
2. We reassign `current` to hold this new reference.

**After: `current = current.next`**
- `current` → node2

As you can see, `current` no longer points to node1; instead, it directly points to node2. This is what we mean when we say "current becomes node2". We're not saying that node2 is created or changed; we're just changing where `current` points in memory.

Now, let's look at how this works in your code example. When we do `node1.next = node2`, we're creating a reference chain:

- `node1.next` → node2
- `node2.next` → node3 (not shown in the diagram, but it's the same idea)

When we call `find_number(node1, 20)`, we start at `node1` and follow the reference chain to find the node with data `20`. By reassigning `current` to point to the next node in the chain, we can traverse the entire list without having to keep track of each node's index.

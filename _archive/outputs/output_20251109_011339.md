**What happens without `next_node`?**


**What happens without `prev`?**

If we don't keep track of the reversed portion with `prev`, we won't know what `current.next` should point to! Let's see what happens if we try to reverse the list without `prev`:

```python

    next_node = current.next
```

But we wanted to reverse the list! Without `prev`, we're not keeping track of the reversed portion, so we can't move forward.

**What happens without `current`?**

If we don't have a position marker (`current`) to traverse the original list, we can't even start reversing it! Let's see what happens if we try to reverse the list without `current`:

As you can see, we get stuck in an infinite loop because we're not moving forward with `current`.

* `prev`: Keeps track of the reversed portion.

Without any of these variables, the algorithm breaks. The three variables are the minimum needed to reverse a linked list.

## 🔑 Key Learning Points

### The Three Variables Are MINIMAL, Not Overcomplicated:

**1. next_node = current.next**
- **Purpose:** Save reference before it's lost
- **Without it:** When current.next = prev, we lose the rest of the list
- **Result:** Can't move forward - stuck!

**2. current.next = prev**
- **Purpose:** Reverse the link
- **Without prev:** What would current.next point to?
- **Result:** Can't build reversed list

**3. prev = current, current = next_node**
- **Purpose:** Move both pointers forward
- **Without proper tracking:** Lose position
- **Result:** Can't traverse

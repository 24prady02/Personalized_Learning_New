# Multi-Turn Conversation Generator

This script generates multi-turn student conversations with **real system analysis** at each turn, using the Groq API to generate responses based on actual analysis results.

## 🎯 Key Features

- ✅ **Real Analysis**: Each turn uses actual orchestrator analysis (HVSAE, Nestor, COKE, CSE-KG, etc.)
- ✅ **Groq API Integration**: Responses generated by Groq API based on real analysis
- ✅ **State Tracking**: Conversation history and student state tracked across turns
- ✅ **Markdown Output**: Formatted output similar to `SYSTEM_OUTPUT_FACTORIAL_WITH_DYNAMIC_LEARNING_STYLE.md`

## 📋 Requirements

1. **Groq API Key**: Set `GROQ_API_KEY` environment variable or add to `config.yaml`
2. **System Initialized**: All models and knowledge graphs must be set up
3. **Dependencies**: All required packages installed

## 🚀 Usage

### Generate 10 Sample Conversations (Quick Start)

```bash
# Set your Groq API key
export GROQ_API_KEY="your_key_here"

# Generate 10 sample conversations automatically
python generate_10_sample_conversations.py
```

This will:
- Generate 10 different conversations (5-6 turns each)
- Cover various programming problems (recursion, arrays, OOP, etc.)
- Use real system analysis for each turn
- Generate Groq responses based on analysis
- Save all conversations to `outputs/` directory
- Create an index file listing all conversations

**Output Files:**
- `outputs/sample_conversation_01.json` - Conversation 1 (JSON)
- `outputs/sample_conversation_01.md` - Conversation 1 (Markdown)
- `outputs/sample_conversation_02.json` - Conversation 2 (JSON)
- ... (and so on for all 10 conversations)
- `outputs/sample_conversations_index.json` - Master index

### Basic Usage

```python
from generate_multi_turn_conversation import MultiTurnConversationGenerator

# Initialize generator
generator = MultiTurnConversationGenerator()

# Define conversation turns
conversation_turns = [
    {
        "turn_number": 1,
        "code": """def factorial(n):
    return n * factorial(n - 1)""",
        "error_message": "RecursionError: maximum recursion depth exceeded",
        "question": "Why is my code giving me a RecursionError?",
        "action_sequence": ["code_edit", "run_test", "run_test"],
        "time_deltas": [15.0, 2.0, 3.0],
        "time_stuck": 20.0
    },
    {
        "turn_number": 2,
        "code": """def factorial(n):
    if n == 0 or n == 1:
        return 1
    return n * factorial(n - 1)""",
        "error_message": "",
        "question": "Great! Can you explain why we need the base case?",
        "action_sequence": ["code_edit", "run_test"],
        "time_deltas": [30.0, 2.0],
        "time_stuck": 0.0
    }
]

# Generate conversation
output = generator.generate_conversation(
    student_id="student_001",
    conversation_turns=conversation_turns,
    output_file="outputs/my_conversation"
)
```

## 📊 Output Format

The script generates two files:

1. **JSON File** (`*.json`): Complete conversation data with all analyses
2. **Markdown File** (`*.md`): Formatted output similar to the factorial example

### Markdown Structure

Each turn includes:

- **Student Input**: Question, code, error, actions
- **System Analysis Pipeline**:
  - STEP 1: HVSAE Multi-Modal Encoding
  - STEP 2: Behavioral Analysis (RNN + HMM)
  - STEP 3: Learning Style Inference
  - STEP 4: Psychological Assessment (Nestor)
  - STEP 5: Knowledge Gap Identification (CSE-KG)
  - STEP 6: Cognitive Assessment
  - STEP 7: Cognitive State Inference (COKE)
  - STEP 8: Intervention Selection
- **System Response**: Groq-generated response based on analysis
- **Metrics**: Quantitative and qualitative metrics

## 🔧 Configuration

### Conversation Turn Structure

Each turn dictionary should contain:

```python
{
    "turn_number": int,              # Turn number (1, 2, 3, ...)
    "code": str,                     # Optional: Student's code
    "error_message": str,            # Optional: Error message
    "question": str,                 # Student's question
    "action_sequence": List[str],    # Actions taken (e.g., ["code_edit", "run_test"])
    "time_deltas": List[float],      # Time between actions (seconds)
    "time_stuck": float              # Total time stuck (seconds)
}
```

## 🔍 Troubleshooting

### Groq API Errors

If you get Groq API errors:
1. Check your API key: `echo $GROQ_API_KEY`
2. Verify API key in `config.yaml`
3. Check API rate limits

### Missing Analysis

If analysis is missing:
1. Ensure all models are initialized
2. Check that knowledge graphs are loaded
3. Verify config.yaml is correct

### Output Not Generated

If output files aren't created:
1. Check write permissions
2. Ensure output directory exists
3. Check for errors in console output

The distinction between `==` and `.equals()` is crucial in Java! 

You're right that `==` compares object references, whereas `.equals()` checks the actual values of the objects. This means when comparing primitive types like `int`, `char`, etc., `==` works as expected because these are not objects.

However, when working with Strings (which you mentioned), even though we use `==` to assign a string literal, it's still an object comparison under the hood. But if we want to compare the actual values of two String objects, we need to use `.equals()`.

Here's an example:

```java
String str1 = "Hello"; // using string literal
String str2 = new String("Hello"); // creating a new String object

System.out.println(str1 == str2); // false because they're different objects
System.out.println(str1.equals(str2)); // true because their values are equal
```

## Ollama response
That's a common confusion! In Java, `&` is a bitwise AND operator, whereas `&&` is a logical AND operator. 

When you use `&`, Java performs a binary operation on the bits of each operand, which isn't what you want when comparing boolean values like true or false.

Try replacing `&` with `&&` in your code and see if that resolves the issue:

```java
if (x > 5 && y < 10) {
    // do something
}
```

This should give you the desired behavior. Remember, `&&` is for logical AND operations, whereas `&` is for bitwise operations!

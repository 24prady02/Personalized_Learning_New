You've discovered a great point about Java's autoboxing feature. For `Integer` objects above 127, `==` doesn't work as expected because these values are not cached in the autobox cache.

In other words, even though you might get the same object reference when comparing two large integers using `==`, this is actually a case of "reference equality" (i.e., both variables point to the same memory location) rather than "value equality".

Here's an example:

```java
Integer x = 128;
Integer y = 128;

System.out.println(x == y); // true, but...
```

In this case, even though `x` and `y` have the same reference value (due to autoboxing), they don't actually point to the same cached object.

To compare values in a robust way, use the `.equals()` method instead:

```java
System.out.println(x.equals(y)); // true
```

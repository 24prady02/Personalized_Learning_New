You're absolutely right about the difference between `==` and `.equals()` when comparing strings in Java!

When you use `==`, it checks if both variables point to the same memory location, which is why you might see "false" even if the strings are identical. This is because a new string object is created each time you assign a value to a String variable.

On the other hand, when you use `.equals()`, it compares the actual character arrays stored in the string objects, not just their references. That's why you get "true" when comparing identical strings using this method.

Here's an example:
```java
String str1 = new String("Hello");
String str2 = new String("Hello");

System.out.println(str1 == str2); // false (same memory location)
System.out.println(str1.equals(str2)); // true (actual character array comparison)
```

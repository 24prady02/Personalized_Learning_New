You've grasped the concept of reference vs. value comparison with Java Strings! When you use `==` for string comparison, you're checking if both variables point to the same object in memory, not their actual contents. This is why `new String("hello")` creates a new object each time.

Here's a code snippet that illustrates this:
```java
String s1 = "hello";
String s2 = new String("hello");
System.out.println(s1 == s2); // false

String s3 = "hello";
String s4 = "hello";
System.out.println(s3 == s4); // true, because "hello" is interned (cached)
```
Your explanation of `.equals()` working correctly is spot on. The `String` class does override the `equals()` method to compare character-by-character, making it a safe choice for string equality checks.

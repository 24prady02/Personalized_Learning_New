When you divide two integers in Java (like 5 and 2), it will indeed give you an integer result, which is just the whole part of the quotient. This is why you're getting 2 instead of 2.5.

To get a decimal result from division, at least one of the numbers being divided must be a floating-point number. You can do this by changing either the dividend (the number being divided) or the divisor (the number doing the dividing) to a double. Here's an example:

```java
double result = 5 / 2; // result will be 2.5
```

In this code, `result` is declared as a `double`, which is a floating-point type that can hold decimal numbers. This tells Java to perform floating-point division when it does the calculation.
